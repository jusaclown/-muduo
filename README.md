## v0.3
加入Channel  
Channel对象负责文件描述符的IO事件分发，它不拥有文件描述符。  
类似pollfd结构体，它也有events_和revents_两个成员，分别表示用户关注的事件和epoll 返回的事件。  
通过在Channel类中设置对应事件的回调函数，我们可以控制当文件描述符fd发生可读/可写等事件时，如何处理这些事件。

> 框架负责从套接字读取数据给用户或者把用户的数据通过套接字发送出去
